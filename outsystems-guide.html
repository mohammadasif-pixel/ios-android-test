1 Overview & Prerequisites

The cordova-plugin-installed-apps plugin detects which payment / UPI wallets (PhonePe, Google Pay, Paytm, BHIM, etc.) are installed on the user’s iPhone. This is an iOS-only plugin — no Android code included.

How it works on iOS

iOS uses UIApplication.canOpenURL: to check if an app is installed. The plugin declares 14 payment app URL schemes in Info.plist via LSApplicationQueriesSchemes. At runtime it checks each scheme and returns only the apps that are actually installed.

2 Step 1 — Host the Plugin

MABS needs to fetch the plugin source at build time. Two options:

Option A — Git Repo (recommended)

Push this repo to GitHub / Bitbucket / GitLab. Make sure plugin.xml is at the repo root.

For private repos, use a Personal Access Token:

https://<TOKEN>@github.com/<org>/cordova-plugin-installed-apps.git

Option B — ZIP Resource

Zip the plugin so plugin.xml sits at the archive root (not inside a subfolder).

Upload the .zip as a Resource in your OutSystems module.

Plugin file structure Your zip or repo root must look like this: 
plugin.xml
package.json
www/
  InstalledApps.js
src/
  ios/
    InstalledApps.h
    InstalledApps.m

3 Step 2 — Extensibility Configuration

Service Studio → your module → Data tab → click module name → right panel → Extensibility Configurations

Single plugin (simplest)

{
  "plugin": {
    "url":  "https://github.com/<your-org>/cordova-plugin-installed-apps.git",
    "name": "cordova-plugin-installed-apps"
  }
}

Multiple plugins

Use the "plugins" array format:

{
  "plugins": [
    {
      "url": "https://github.com/<org>/cordova-plugin-installed-apps.git"
    },
    {
      "url": "https://github.com/other/other-plugin.git"
    }
  ]
}

Using ZIP resource

{
  "plugin": {
    "url": "resource://cordova-plugin-installed-apps.zip"
  }
}

The resource:// prefix tells MABS to look in the module’s Resources folder.

OutSystems 11 Note Extensibility Configurations only accepts valid JSON. If malformed, Service Studio silently ignores it and plugin won’t install. Validate your JSON before pasting.

4 Step 3 — Set MABS Version

This plugin uses the async $resolve pattern in JavaScript, which requires MABS 10+.

Option A — Force via Extensibility

{
  "plugin": {
    "url": "https://github.com/<org>/cordova-plugin-installed-apps.git"
  },
  "preferences": {
    "global": [
      { "name": "mabs-version", "value": "10.0" }
    ]
  }
}

Option B — Check in LifeTime

LifeTime → Environments → your environment → Mobile Apps Build Service

Confirm MABS version is 10.0 or higher.

Stuck on MABS 9? The $resolve async pattern won’t work. See the MABS 9 fallback in Step 6 below.

7 Step 6 — JavaScript Node Setup

Double-click the JavaScript node (CallPlugin). In the right panel:

6a — Add Output Parameters to JS Node

NameData Type
AppsJsonText
ErrorMsgText

6b — JavaScript Body (MABS 10+)

Copy-paste this exactly into the JavaScript editor:

var done = $resolve;   // async pattern — MABS 10+ / ODC

// Guard: plugin only exists on native device builds
if (!window.cordova || !cordova.plugins || !cordova.plugins.InstalledApps) {
    $parameters.AppsJson = "[]";
    $parameters.ErrorMsg = "Plugin not available — run on a real device, not Preview in Browser.";
    return done();
}

cordova.plugins.InstalledApps.getPaymentApps(
    function (apps) {
        $parameters.AppsJson = JSON.stringify(apps || []);
        $parameters.ErrorMsg = "";
        done();
    },
    function (err) {
        $parameters.AppsJson = "[]";
        $parameters.ErrorMsg = String(err || "unknown error");
        done();
    }
);

6c — MABS 9 Fallback (if you cannot upgrade)

MABS 9 — synchronous JS node On MABS 9, $resolve does not work. Use client variables + timer delay instead.

Step 1: Create two Client Variables at module level:

NameData Type
CV_PaymentAppsJsonText
CV_PaymentAppsErrorText

Step 2: Use this JavaScript body:

// MABS 9 sync fallback
if (!window.cordova || !cordova.plugins || !cordova.plugins.InstalledApps) {
    $parameters.AppsJson = "[]";
    $parameters.ErrorMsg = "Plugin not available.";
    return;
}

cordova.plugins.InstalledApps.getPaymentApps(
    function (apps) {
        $public.ClientVariables.CV_PaymentAppsJson = JSON.stringify(apps || []);
        $public.ClientVariables.CV_PaymentAppsError = "";
    },
    function (err) {
        $public.ClientVariables.CV_PaymentAppsJson = "[]";
        $public.ClientVariables.CV_PaymentAppsError = String(err || "unknown");
    }
);

$parameters.AppsJson = "[]";
$parameters.ErrorMsg = "loading";

Step 3: After calling GetPaymentApps, add a Timer delay (500ms), then read from Client.CV_PaymentAppsJson and re-run JSONDeserialize.

9 Step 8 — Bind to Screen

Call from OnReady

Screen → Events → OnReady handler

  1. Drag GetPaymentApps into the OnReady flow.
  2. Add an If node: GetPaymentApps.ErrorMsg = ""
  3. True branch: Assign local variable PaymentAppsList (type PaymentApp List) = GetPaymentApps.Apps
  4. False branch: Show feedback message with error.

Display in a List

Drag a List widget onto the screen. Set Source = PaymentAppsList. Inside the list item:

WidgetBinding
ImageCurrent.PaymentApp.Icon (Type: External URL)
Expression (name)Current.PaymentApp.AppName
Expression (scheme)Current.PaymentApp.UrlScheme

10 Step 9 — iOS Certificates in LifeTime

LifeTime → Applications → your app → Native Platforms → Configure iOS App

What you need from Apple Developer Portal

ItemWhere to get itFormat
Distribution CertificateCertificates, IDs & Profiles → Certificates.p12 + password
Provisioning ProfileProfiles → iOS App Development or Distribution.mobileprovision

Upload in LifeTime

  1. Click Configure iOS App
  2. Upload the .p12 file and enter its password
  3. Upload the .mobileprovision file
  4. Set Bundle Identifier to match your provisioning profile (e.g. com.magentamobility.yourapp)
  5. Click Save
Critical The provisioning profile must include LSApplicationQueriesSchemes entitlement support. Standard Distribution and Ad Hoc profiles support this by default. No special entitlements needed.

11 Step 10 — Generate iOS Build

LifeTime → Applications → your app → Native Platforms → Generate iOS App

  1. Click Generate iOS App
  2. MABS will:
    • Fetch plugin from Git URL / ZIP resource
    • Merge 14 URL schemes into Info.plist (LSApplicationQueriesSchemes)
    • Compile InstalledApps.h + InstalledApps.m (Objective-C)
    • Sign the IPA with your certificate
  3. Wait for build to complete (6–12 minutes first build)
  4. Download the .ipa file

Install the IPA

TestFlight (App Store profile)

Upload IPA to App Store Connect → TestFlight → invite testers.

Ad Hoc

Distribute via Diawi, AppCenter, or direct install on registered devices.

Build log If build fails, check MABS log: LifeTime → Native Platforms → Download Build Logs. Search for cordova-plugin-installed-apps.

iOS-Specific Notes

LSApplicationQueriesSchemes

Apple caps URL scheme whitelist at 50 entries. This plugin uses 14. Count other plugins’ schemes before submitting to App Store.

Declared schemes:

phonepe, gpay, tez, paytm, paytmmp, bhim, amazon, amazonpay,
whatsapp, cred, mobikwik, freecharge, airtel, jiopay

How Detection Works

iOS uses UIApplication.canOpenURL:. Returns true only if:

  1. URL scheme is listed in LSApplicationQueriesSchemes (handled by plugin.xml)
  2. Target app is actually installed on device
  3. Build is properly signed

Privacy Manifest (iOS 17+)

canOpenURL: is not a tracked API under Apple’s privacy-manifest requirements. No PrivacyInfo.xcprivacy changes needed.

App Store Review

If Apple asks why you query payment schemes:

Suggested response “We query UPI payment app URL schemes to display only the wallets the user already has installed, providing a streamlined checkout experience. We do not collect or transmit any data about installed applications.”

Troubleshooting

SymptomCause & Fix
cordova.plugins.InstalledApps is undefined Running in Preview in Browser. Plugin is native-only. Deploy to real device. JS guard in Step 6 handles this gracefully.
Empty list on device with payment apps installed Build not signed with correct profile. canOpenURL: returns false for unsigned/debug builds. Rebuild with distribution certificate.
MABS error: “Plugin doesn’t support cordova-ios” MABS version too old. Add mabs-version: 10.0 preference (Step 3).
MABS error: “Failed to fetch plugin” Git URL unreachable. Check: repo is public (or token URL for private), plugin.xml at repo root, URL ends in .git.
JS node returns immediately with empty data (MABS 9) $resolve not supported. Use MABS 9 fallback (Step 6c).
JSON parse error in JSONDeserialize Plugin returned unexpected format. Log AppsJson value. Common cause: plugin not installed (returns undefined).
App Store reject: “Non-public API” Not caused by this plugin — only public UIApplication APIs used. Check other plugins.
Duplicate LSApplicationQueriesSchemes warning Another plugin also declares URL schemes. Cordova merges them — warning is safe to ignore.

API Reference

getPaymentApps(successCallback, errorCallback)

Returns only UPI / wallet apps from the built-in catalog. Recommended for checkout flows.

getInstalledApps(successCallback, errorCallback)

Returns every app whose URL scheme is declared in LSApplicationQueriesSchemes.

getAppInfo(bundleId, successCallback, errorCallback)

Detailed info for a single app. On iOS, privacy restrictions limit this to the host app’s own bundle ID only.

Response Shape (getPaymentApps array element)

{
  "appName":     "PhonePe",
  "packageName": "com.phonepe.app",       // from catalog (for reference)
  "urlScheme":   "phonepe",               // iOS URL scheme used for detection
  "isSystemApp": false,
  "icon":        "https://cdn-icons-png.flaticon.com/512/825/825454.png"
}

Quick Checklist

  • Plugin hosted on reachable Git URL or uploaded as ZIP resource
  • Extensibility Configurations JSON pasted in Service Studio module properties
  • MABS version set to 10.0+ (via preference or LifeTime)
  • PaymentApp structure created with 5 attributes
  • GetPaymentApps client action created with JS node + JSONDeserialize + Assign
  • Screen calls GetPaymentApps from OnReady and binds to List widget
  • iOS Distribution Certificate (.p12) uploaded in LifeTime
  • Provisioning Profile (.mobileprovision) uploaded in LifeTime
  • Bundle Identifier matches provisioning profile
  • iOS build generated successfully in MABS
  • Tested on real iPhone with at least one payment app installed
  • Tested Preview in Browser fallback (graceful error, no crash)
Done! Plugin is iOS-only. Once you push to Git and paste the Extensibility JSON in Service Studio, MABS handles the native compile. The key steps: host plugin → paste JSON → create structure + client action → upload certificates → build → test on real iPhone.
